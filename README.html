Expression Syntax and Capabilities Documentation

Basic Arithmetic Operations

The calculator supports standard arithmetic operations:

  • Addition: +
  • Subtraction: -
  • Multiplication: *
  • Division: /
  • Modulo: %
  • Exponentiation: ^

Example: 2 + 3 * 4 - 5 / 2

Numbers

Decimal Numbers

Standard decimal notation is supported.

Example: 3.14159

Integer Bases

The calculator supports numbers in different bases:

  • Binary (base 2): Prefix with 0b

    • Example: 0b1010 (equals 10 in decimal)

  • Octal (base 8): Prefix with 0

    • Example: 012 (equals 10 in decimal)

  • Hexadecimal (base 16): Prefix with 0x or 0X

    • Example: 0xA or 0XA (equals 10 in decimal)

Mathematical Constants

The following mathematical constants are supported:

  • Pi: Use pi
  • Euler's number: Use e

Example: pi * r^2

Functions

The calculator supports various mathematical functions:

  • Square root: sqrt(x)
  • Natural logarithm: log(x)
  • Random number generation: rand()

Example: sqrt(16) + log(10)

Variables and Assignments

You can define and use variables in expressions:

  • Assignment: Use =
  • Variable names can contain letters, numbers, and underscores, but must start with a letter or underscore

Example: x = 5; y = x + 3; y * 2

Conditional Statements

The calculator supports basic if-else conditionals:

if (condition) {
    // expressions
} else {
    // expressions
}

Example:

if (x > 5) {
    y = x * 2
} else {
    y = x / 2
}

Function Definitions

You can define custom functions:

def function_name(param1, param2, ...) {
    // function body
    return expression
}

Example:

def square(x) {
    return x * x
}

Comments

The calculator supports two types of comments:

  1. Line comments: Start with 0* and end with *0

    2. Example: 0* This is a comment *0

  2. String comments: Use 0*string(...) format

    3. Example: 0*string(This is a string comment)

Precision

The calculator supports high precision calculations up to 100,000 digits.

Order of Operations

The calculator follows standard mathematical order of operations (PEMDAS):

  1. Parentheses
  2. Exponents
  3. Multiplication and Division (left-to-right)
  4. Addition and Subtraction (left-to-right)

Error Handling

The calculator will return an error message for invalid expressions, such as:

  • Division by zero
  • Invalid function calls
  • Syntax errors
  • Undefined variables or functions
